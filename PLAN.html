ClaudeChat 跨网络远程访问 & iPhone 支持

使 Mac App 在开机状态下随时可通过 iPhone（或其他设备）浏览器访问，支持查看会话、发送消息、接收回复，无需外部服务器、无需 Apple Developer、完全免费。

前提：Mac 必须开机且 App 后台运行。所有 Claude 调用仍然由 Mac 上的 claude CLI 执行。iPhone 只是远程输入输出终端。

2. 安全模型

2.1 三层防护

层级机制防护目标
1. URL 随机性cloudflared 生成随机子域名
https://happy-random-fox.trycloudflare.com		暴力扫描
2. 邀请码认证用户输入 8 位邀请码 → 服务端验证 → 签发 JWT未授权访问
3. TLS 加密Cloudflare 自动 HTTPS 证书中间人攻击

2.2 邀请码机制

Mac App 端：设置面板中生成/查看/撤销邀请码。每个邀请码存储于本地，支持设置备注（如"我的手机"）和过期时间。

iPhone 端：首次访问 Web 页面 → 显示登录界面 → 输入邀请码 → 后端验证成功 → 返回 JWT（存储于浏览器 localStorage，24h 有效）→ 进入聊天界面。

验证流程：

POST /api/login { "code": "A1B2C3D4" } → 服务端查询 UserDefaults 中的邀请码列表 → 匹配成功 → 生成 JWT (HS256, userId + exp + iat) → 返回 { "token": "eyJhbG...", "expiresIn": 86400 } 后续请求 → Authorization: Bearer <token> → 验证 JWT → 放行

2.3 邀请码管理面板（Mac 原生设置）

操作说明
生成邀请码随机 8 位字母数字组合，默认 7 天过期
查看列表显示所有邀请码、备注、创建时间、过期时间、使用次数
撤销立即失效，该码无法再登录（已有 JWT 可继续用至过期）
续期延长有效期

4. 数据流

── iPhone 发消息 ────────────────────────────────────────── iPhone 输入 "帮我写一个排序函数" → POST /api/send { sessionId, message } → 中间件验证 JWT → SessionStore 找到对应 Session → ChatViewModel.send(message) → ClaudeExecutor.run() → Process: claude -p "帮我写一个排序函数" --resume sid → stdout stream-json 逐行解析 → POST /api/send 响应: { conversationId } (立即返回) ── 获取回复 (轮询) ──────────────────────────────────────── iPhone 每 2 秒轮询 → GET /api/stream/{conversationId} → 返回该次对话的增量文本 → 对话完成后标记 completed=true

8. 实施顺序

  1. Package.swift 依赖 + Hummingbird 最小原型

    新增 hummingbird、jwt-kit 依赖。创建 HTTPServer.swift，启动 localhost:8080，返回 "Hello"。验证 Swift build。

  2. 邀请码认证系统

    InviteCodeManager.swift（生成/查询/撤销/持久化到 UserDefaults）+ AuthMiddleware.swift（JWT 签发和校验）。提供 POST /api/login 端点。

  3. 会话 & 消息 REST API

    SessionStore 与 HTTP 服务器桥接。GET /api/sessions、GET /api/sessions/:id/messages、POST /api/sessions/:id/send（触发 claude CLI 调用，结果通过 /stream 轮询获取）。

  4. Web 聊天界面

    index.html SPA：登录表单 → 会话列表侧边栏 → 聊天气泡 + Markdown 渲染 + 流式文本追加。Tailwind CSS CDN 响应式布局。

  5. cloudflared 隧道集成

    TunnelManager.swift：检测/安装 cloudflared → 启动隧道 → 监控进程 → 获取公网 URL → 显示在菜单栏。

  6. MenuBarExtra + App 生命周期

    App.swift 改为 MenuBarExtra 入口。Dock 不再显示图标，点菜单栏打开聊天窗口。关闭窗口不退出 App，只在菜单栏点"退出"时停止。

  7. 设置面板扩展

    SettingsView 新增邀请码管理页（生成/查看/撤销/续期）+ 隧道配置（启动/停止/URL 复制）。

  8. 端到端测试

    iPhone Safari 打开隧道 URL → 输入邀请码 → 查看会话 → 发送消息 → 看到流式回复。

10. 安全注意事项

  • 邀请码使用 SHA256 哈希 存储，不存明文
  • JWT 密钥为 随机 32 字节，首次启动时生成，存于 Keychain
  • cloudflared 不在日志中暴露请求内容
  • 非邀请码的 API 路径全部走 JWT 校验中间件
